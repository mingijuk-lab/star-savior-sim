# 🚀 스타 세이비어 엔진 v5.0 유지보수 및 확장 가이드

이 문서는 엔진 표준화 작업(v5.0)을 통해 변경된 핵심 아키텍처와 데이터 관리 규칙을 기록하여, 향후 개발 시 정합성을 유지하고 휴먼 에러를 방지하기 위해 작성되었습니다.

---

## 🏗️ 1. 핵심 아키텍처: 공용 버프 시스템 (Non-Stacking)

엔진 v5.0부터는 모든 공용 버프를 `common_buffs` 딕셔너리로 관리하며, **중첩되지 않고 최대치 하나만 적용**되는 '비중첩 원칙'을 준수합니다.

### 🔹 구현 방식
```python
# 엔진 내부 (예시)
common_buffs = {
    "atk": t.get("atk_buf", 0.0),
    "def_red": max(t.get("def_red_buf", 0.0), 0.30 if smile_def_red_turns > 0 else 0.0),
    ...
}
# 캐릭터 고유 공증 트리거 반영
if charles_atk_buff_turns > 0 or (is_lin and lin_atk_buff_turns > 0):
    common_buffs["atk"] = max(common_buffs["atk"], 0.30)
```

### ⚠️ 주의사항
- **새로운 공용 버프 추가 시**: `common_buffs` 선언부에서 `max()`를 사용하여 기존 수치와 경합하도록 작성하십시오.
- **고유 버프(Unique)와의 분리**: 린의 '명월'이나 샤를의 '럭키토큰'처럼 일반 버프와 별개로 작동해야 하는 효과는 `common_buffs`에 넣지 말고 `dyn_atk_p_sum` 등에 직접 합산하십시오.

## 🧪 2. 캐릭터별 특수 로직 구현 패턴

### 🔹 린 (Lin): 스택 및 상태 기반 시스템
린과 같은 복잡한 캐릭터는 '상태(Turns)'와 '스택'을 분리하여 관리합니다.
- **Leap 스택**: 행동 시작 시 `leap_gain`을 결정하여 합산 (범용 +1, 특수 조건 +3).
- **명월/청풍 상태**: `lin_bright_moon_turns`, `lin_cheongpung_turns` 변수로 관리하며, 턴 종료 시(`lin_rem_turns > 0`) 1씩 차감합니다.

### 🔹 로자리아 (Rosaria): 추가타 판정 로직
최근 최적화된 로자리아의 추가타 로직은 '준비 상태' 플래그를 사용합니다.
- `ros_can_extra`: 특수기 사용 시 업화 3스택 달성 시 `True`로 전환.
- `is_extra_attack` 진입 시 `ros_can_extra`가 `False`면 해당 턴을 건너뜁니다(`continue`).
- **목표**: 로테이션 데이터에 추가타가 명시되어 있더라도 엔진 내 스택 조건이 맞지 않으면 실행되지 않도록 강제합니다.

## 🧹 3. 데이터 정화 및 명명 규칙

### 🔹 데이터 오염 방지 (Purification)
- **일반 샤를(Striker) 제거**: 기존의 누적된 오염 데이터를 방지하기 위해 일반 샤를 모델은 완전히 폐기되었습니다. 신규 캐릭터 추가 시 반드시 `characters.json`의 기존 키 값을 확인하십시오.
- **이름 정규화**: `is_lin = "린" in cname_norm`과 같이 괄호나 하이픈을 제거한 정규화된 이름으로 캐릭터를 식별하여, `린(바니걸)` 등 파생형과의 호환성을 확보하십시오.

### 🔹 사이클 표기 표준화
- **B (Basic)**: 기본기
- **S (Special)**: 특수기
- **U (Ultimate)**: 궁극기
- **TN+ (Extra Attack)**: 시간 0소모, 쿨다운 비소모 추가 공격
- **TN+1 (Extra Turn)**: 시간 소모, 쿨다운 1회 소모 추가 턴
