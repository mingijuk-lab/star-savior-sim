# 스타 세이비어 — 통합 데미지 시뮬레이션 가이드라인 (v29.0)

> 본 문서는 데미지 시뮬레이션 엔진(`Core/calc_engine_v5.py`)의 **절대적 기준이 되는 통합 문서**입니다.
> 5개 여정(Journey) 조합 최적화 엔진, 축복(Blessing) 통합 시스템, 동적 패시브 엔진, 그리고
> 프레이 달속성파티 HR/냉각 독립 스택, DEF Penetration, AoE 동적 판정을 포함합니다.
>
> **최종 갱신**: 2026-04-03

## 1. 전투 및 시스템 기본 규칙

### 1.1 행동 및 속도
- **행동 주기**: `1000 / 최종_속도`
- **행동 게이지(AG)**: 100% 도달 시 행동. 스킬 효과로 인한 AG 증가는 소요 시간을 감소시킴.
- **추가 턴**: AG 100% 즉시 충전 (소요 시간 0).
- **AG 감소 캐리**: `ga_red_carry` 변수로 다음 턴의 AG 감소율을 전달 (합산 `+=` 방식).

### 1.2 스킬 우선순위 및 쿨타임
- **우선순위**: `궁극기 > 특수기(조건 충족 시) > 기본기`
- **쿨타임**: N턴 쿨타임 스킬은 사용 후 N-1번의 다른 행동이 필요함.
- **예외**: 추가턴 제공 특수기(프레이, 에핀델, 샤를(바니걸))는 궁극기보다 우선 사용.

### 1.3 추가 액션 규칙 (Extra Action Rules)
- **추가타 (Extra Attack, TN+)**:
    - 트리거 조건 충족 시 즉시 발생.
    - **시간 소모 0 (AG 무관)**, **스킬 쿨다운 감소/갱신 없음**.
    - 예: 클레어 냉각 5회 시 특수기 트리거, 로자리아 업화 3회 추가 기본기.
    - JSON에서 `"is_extra_attack": true`로 마킹.
- **추가턴 (Extra Turn, TN+1)**:
    - 스킬 효과로 즉시 다음 행동 획득.
    - **시간 소모 0** (`is_extra_turn: true`), **스킬 쿨다운은 정상 감소**.
    - 예: 프레이 특수기, 에핀델 특수기, 샤를(바니걸) 특수기.

### 1.4 버프 적용 타이밍 (Buff Timing)
- **지연 적용 (Delayed Apply)**: 스킬 사용 시 발생하는 자가 버프(예: 공격력 증가, 치확 증가)는 해당 턴의 데미지 계산에는 반영되지 않으며, **다음 행동(턴)부터 적용**됩니다.
- **동적 패시브 누적 (v15.0+)**: 캐릭터 고유 패시브는 턴 시작, 행동 시작, 또는 타격 시점에 실시간으로 누적됩니다. 상시 최대 스택 가정(Steady-state)은 **금지**됩니다.
- **AX 초기화**: 아르카나(AX) 스택은 **궁극기 사용 시 0으로 초기화**되며, 이는 다음 턴부터 공격력 감소 수치로 반영됩니다. (궁극기 타격 자체는 5스택 상태로 적용)

### 1.5 보편적 속성(Attribute) 스택 — 냉각/격동/점화/도약
- **기본 규칙**: 모든 캐릭터는 **기본기(Basic Attack)** 사용 시 '속성(attr)' 스택이 **+1** 적립됩니다 (최대 5회).
- **특수 적립**: 프레이/에핀델 특수기 → `+3`, 샤를(바니걸) 특수기 → `+3`
- **5스택 리셋 조건**: 프레이(냉각 5→리셋), 클레어(냉각 5→추가타+리셋)
- **캐릭터별 고유 스택과의 관계**: 속성 스택은 로자리아 업화, 프레이 HR 등과 별개로 독립 추적됩니다.

### 1.6 방어 계산 (v14.2+)
- **기본 방어력**: 타겟 3000
- **방어율**: `effective_def / (effective_def + 3000)`
- **유효 방어력**: `target_def × (1 - 방감률) × (1 - 관통률)`
- **방어력 감소(DEF Reduction)**: 스마일 궁극기 → 30% 감소 (2턴)
- **방어력 관통(DEF Penetration)**: 캐릭터/여정의 `DEF_PEN` 스탯 + 턴 데이터의 `def_pen_buf`

## 2. 5개 여정(Multi-Journey) 최적화 시스템 (v14.0+)

축복(Blessing, 구 아르카나)과 여정(Journey)을 통합하여 5개 여정 슬롯 + 1개 축복 슬롯으로 최적의 조합을 찾아냅니다.

### 2.1 조합 최적화 (Combinatorial Optimization)
- 엔진은 `Data/equipments.json`의 `journeys` 섹션에서 캐릭터/클래스 제한을 통과한 모든 유효 여정을 대상으로 **5개를 선택하는 모든 조합(nCr)**을 시뮬레이션합니다.
- 각 조합에 대해 축복(`AX`, `FX`, `EX`, `아르카나_속도`, `None`)을 추가로 순회합니다.
- 결과적으로 조합 수 = `C(n,5) × 5가지 축복` 전수 조사.

### 2.2 장비 세트 전수 탐색
- 4세트 × 2세트의 모든 조합(현재 12가지)에 대해 개별적으로 최적화합니다.
- 각 장비 세트에서 Standard(궁극기 사용) / No-Ult(궁극기 미사용) 최적 빌드를 각각 산출합니다.

### 2.3 장착 제한 및 특수 로직

| 조건 유형 | 예시 | 동작 |
|----------|------|------|
| `restrict.class` | `"class": "레인저"` | 해당 클래스만 여정 풀에 포함 |
| `restrict.char` | `"char": "유미나"` | 해당 캐릭터 전용 |
| 어쌔신 필수 여정 | `키라만큼 귀여워` | 어쌔신 클래스는 반드시 포함, 나머지 4개 조합 탐색 |

### 2.4 주요 여정(Journey) 스택 및 특수 로직

| 여정 이름 | 적용 대상 | 엔진 변수 | 상세 사양 |
| :--- | :---: | :--- | :--- |
| **깊은 애도** | 레인저 | `ra_cr_stack` | 기본기/추가타 시 치확 +5% (최대 5회, 총 25%) |
| **하늘의 심판** | 캐스터 | `caster_cd_stack` | 매 행동 시작 시 치피 +10% (최대 3회, 총 30%) |
| **키라만큼 귀여워** | 어쌔신 | `assassin_spd_stack` | 매 행동 시작 시 속도 +10 (최대 3회, 총 30) |
| **허수의 개척자** | 공통 | `omega_dmg_stack` | 특수기 사용 시 DI +5% (최대 5회, 총 25%). 보너스도 특수기에만 적용. |

## 3. 하이브리드 장비 시스템 (v13.0+)

### 3.1 4+2 세트 조합 (Dynamic Combinations)
- **4세트**: 공격(ATK 20%), 통찰(CR 30%), 파괴(CD 40%), 체력(HP 30%)
- **2세트**: 투지(ATK 5%), 장벽(HP 10%), 보조(Stat 없음)
- 총 12가지 조합이 `data_loader_v5.py`의 `load_combined_equipments()`에서 자동 생성됩니다.

### 3.2 부옵션 변수 관리 ($Var$)
- 모든 장비 6부위에 동일한 부옵션이 `$ATK$`, `$SPD$`, `$CR$`, `$CD$` 변수로 적용됩니다.
- 대화형 프롬프트(`get_interactive_substats()`)로 입력하거나, `automated_run.py`에서 고정값 주입.
- **시스템 고정치**: `ATK% +17.25%` (부옵션 16.25% + 전술 1%), `SPD +60`, `ATK FLAT +1000`

## 4. 공격력 및 데미지 공식

### 4.1 실효 공격력 공식 (v14.3+ Unified Summation)
```
raw_pool    = 기본_공격력 + 1250 (StellaArchive)
static_atk% = 장비세트% + 여정% + 부옵션%(0.1725)
eff_base    = raw_pool × (1 + static_atk%) + 1000 (SystemConstant)

Σ_dynamic   = AX_stack × 0.08
            + JSON_turn_atk_buf
            + 리디아_stack × 0.06
            + 유미나_stack × 0.04 (1lv: 0.02)
            + 피의메아리_stack × 0.03
            + ...

resonance   = raw_pool × res% + res_flat

eff_atk     = eff_base × (1 + Σ_dynamic) + resonance
```

### 4.2 최종 데미지 (Defense-Integrated)
```
effective_def = target_def × (1 - def_red) × (1 - def_pen)
def_mult      = 1 - effective_def / (effective_def + 3000)

total_DI = 스킬_DI + Σ(여정_DI) + omega_boost + fx_carry + 점화DI + ...
hr_prob  = 1.0 (프레이 HR ≥ 5 && 기본기) else 0.0

Damage = (eff_atk × coeff × (1 + total_DI + CR × CD + hr_prob)) × def_mult
```

### 4.3 DPS 계산
```
DPS = Σ(turn_damage) / Σ(turn_time)
turn_time = 1000 / final_speed × (1 - AG_reduction)  # 추가턴은 0
```

### 4.4 Max Hit (Peak) 계산
```
Peak = (eff_atk × coeff × (1 + total_DI + 1.0(CD) + 1.0(HR))) × def_mult
```

## 6. 동적 전체 공격(AoE) 판정 시스템 (v26.0+)

### 6.1 판정 기준
- `characters.json`의 스킬 `"부가"` 필드에 `"전체 공격"` 문자열이 포함되면 `is_aoe=True`.
- **유미나 카탈리스트 예외**: `경력직 용병` 여정 장착 시 기본기를 AoE로 강제 판정.

### 6.2 AoE 캐릭터 시뮬레이션 분기 (v27.0+)
- AoE 스킬 보유 캐릭터(유미나 제외)는 **보스1인(target_count=1)** / **일반3인(target_count=3)** 두 가지로 자동 분기.
- 유미나는 AoE이지만 분기하지 않음 (대신 카탈리스트 로직으로 처리).

### 6.3 피의 메아리 스택 가속
- AoE + 치명타 발생 시 `target_count`만큼 `rosa_atk_stack` 누적 (+3/타겟, 캡 5).
- 3인 환경에서 1회 크리티컬로 3스택 획득 → 예열 시간 대폭 단축.

## 7. 빌드 진화 경로 (Scaling Profile, v19.0+)

### 7.1 개요
- 부옵션 0%부터 50%까지 12.5% 간격으로 스윕하며, 각 지점에서 **전 장비세트 × 전 여정 × 전 축복을 전수 재탐색**하여 절대적 1위 빌드를 찾습니다.
- `gear_sensitivity_v5.py`의 `profile_stat_scaling()` 함수가 이를 수행합니다.

### 7.2 추적 스탯
- **공격력%** (`StatType.ATK`), **치명타 피해** (`StatType.CRIT_DAMAGE`), **치명타 확률** (`StatType.CRIT_RATE`)

### 7.3 전환점 포착 기준
- 장비세트, 축복, 또는 여정 조합 중 **하나라도 변경**되면 전환점으로 기록.
- 각 전환점에 DPS 수치와 MaxHit 수치를 함께 표기.

1. `Data/characters.json`에 캐릭터 데이터 추가 (또는 스펙 마스터 → extract)
2. `Tools/update_rotations.py`에 해당 캐릭터의 스택/트리거 로직 추가
3. `Core/calc_engine_v5.py`의 `calculate_dps()`에서:
   - 캐릭터 감지 (`"캐릭터명" in cname`)
   - 고유 스택 변수 초기화 (L169~L206 부근)
   - 매 턴 스택 업데이트 로직 추가
   - 스택 효과의 데미지 반영 (DI, ATK%, CR, CD 등)

   - `any(j.name == "여정이름" for j in jrs)` 패턴으로 감지
   - 스택 변수 추가 및 턴별 업데이트 로직 구현

### 새 축복 추가 시
