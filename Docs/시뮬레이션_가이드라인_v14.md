# 스타 세이비어 — 통합 데미지 시뮬레이션 가이드라인 (v14.4)

> 본 문서는 데미지 시뮬레이션 엔진(v14.0+)의 **절대적 기준이 되는 통합 문서**입니다.
> 5개 여정(Journey) 조합 최적화 엔진, 아르카나-여정 통합 시스템, 그리고 제한 조건 자동 필터링을 포함합니다.

---

## 1. 전투 및 시스템 기본 규칙

### 1.1 행동 및 속도
- **행동 주기**: `1000 / 최종_속도`
- **행동 게이지(AG)**: 100% 도달 시 행동. 스킬 효과로 인한 AG 증가는 소요 시간을 감소시킴.
- **추가 턴**: AG 100% 즉시 충전 (소요 시간 0).

### 1.2 스킬 우선순위 및 쿨타임
- **우선순위**: `궁극기 > 특수기(조건 충족 시) > 기본기`
- **쿨타임**: N턴 쿨타임 스킬은 사용 후 N-1번의 다른 행동이 필요함.

### 1.3 추가 액션 규칙 (Extra Action Rules)
- **추가타 (Extra Attack, TN+)**: 
    - 트리거 조건 충족 시 즉시 발생. 
    - **시간 소모 0 (AG 무관)**, **스킬 쿨다운 감소/갱신 없음**.
    - 예: 클레어 냉각 5회 시 **강화 평타가 아닌 특수기** 트리거, 로자리아 업화 3회 트리거.
- **추가턴 (Extra Turn, TN+1)**: 
    - 스킬 효과로 즉시 다음 행동 획득. 
    - **시간 정상 소모**, **스킬 쿨다운 1턴 감소**.
    - 예: 프레이 특수기 사용 시.

### 1.5 버프 적용 타이밍 (Buff Timing)
- **지연 적용 (Delayed Apply)**: 스킬 사용 시 발생하는 자가 버프(예: 공격력 증가, 치확 증가)는 해당 턴의 데미지 계산에는 반영되지 않으며, **다음 행동(턴)부터 적용**됩니다.
- **AX 초기화**: 아르카나(AX) 스택은 **궁극기 사용 시 0으로 초기화**되며, 이는 다음 턴부터 공격력 감소 수치로 반영됩니다. (궁극기 타격 자체는 5스택 상태로 적용)

### 1.4 보편적 속성(Attribute) 스택
- **기본 규칙**: 모든 캐릭터는 **기본기(Basic Attack)** 사용 시 '속성' 스택이 **+1** 적립됩니다 (최대 5회). 턴 당 1회씩만 적용됨.(예: 샤를(바니걸)처럼 기본기 2번 사용 시 2회 적립되는 것이 아니라 1회만 적립됨)
- **특수**: 캐릭터별 고유 스택(냉각, 격동, 점화 등)과 별개로 동작할 수 있습니다. (예: 로자리아 업화, 프레이 HR )

## 2. 5개 여정(Multi-Journey) 최적화 시스템 (v14.0)

아르카나(Arcana)와 여정(Journey)의 구분을 없애고, 모든 버프를 5개의 여정 슬롯으로 통합하여 최적의 조합을 찾아냅니다.

### 2.1 조합 최적화 (Combinatorial Optimization)
- 엔진은 `Data/equipments.json`에 정의된 전체 여정 중 캐릭터/클래스 제한을 통과한 모든 유효 여정을 대상으로 **5개를 선택하는 모든 조합**을 시뮬레이션합니다.
- 약 1,200~1,300개 이상의 조합을 전수 조사하여 각 장비 세트별 최적의 구성을 도출합니다.

### 2.2 장착 제한 및 특수 로직
- **제한 조건**: 특정 캐릭터(예: 키라) 혹은 클래스(예: 캐스터) 전용 여정은 해당 조건에 맞는 경우에만 조합 풀에 포함됩니다.
- **기간 한정 버프**: "최초 2턴간 치확 30%"와 같은 한정적 버프는 시뮬레이션의 `action_idx`를 기반으로 정확한 지속 시간 동안만 적용됩니다.
- **스택형 버프**: 턴 시작 시 또는 특정 행동 시 중첩되는 스탯(속도, 치피, 치확 등)은 중첩 제한까지 실시간 반영됩니다.

### 2.4 주요 여정(Journey/Arcana) 스택 및 특수 로직
- **깊은 애도 (레인저)**: 기본기 사용 시 치명타 확률 +5% 중첩 (최대 **5회**, 총 25%).
- **하늘의 심판 (캐스터)**: 매 행동 시작 시 치명타 피해 +10% 중첩 (최대 3회, 총 30%).
- **키라만큼 귀여워 (어쌔신/키라)**: 매 턴 시작 시 속도 +10 중첩 (최대 3회, 총 30). (예: T1에서 추가턴 특수기 사용시, T1 + 10, T2(추가턴) + 10, T3 + 10)
- **허수의 개척자 (오메가)**: 특수기 사용 시 **해당 특수기 피해량(DI)** +5% 중첩 (최대 5회, 총 25%).
- **연쇄 피해 (Chain Damage)**: 
    - **유미나 (카탈리스트)**: 치명타 시 공격력의 5% 추가 피해.
    - **어쌔신 (서투른 욕망 해소법)**: v14.0 엔진에서 계산 제외 (표준화).

## 3. 하이브리드 장비 시스템 (v13.0+)

### 3.1 4+2 세트 조합 (Dynamic Combinations)
- **4세트**: 공격(ATK 20%), 통찰(CR 30%), 파괴(CD 40%), 체력(HP 30%)
- **2세트**: 투지(ATK 5%), 장벽(HP 10%), 보조(Stat 없음)

### 3.2 부옵션 변수 관리 ($Var$)
- 모든 장비 부위의 부옵션은 시뮬레이션 시작 시 대화형 프롬프트를 통해 공%, 속도, 치확, 치피 수치를 입력받습니다.
- **균등 적용**: 입력한 수치는 모든 6부위에 동일하게 적용됩니다.

## 4. 공격력 및 데미지 공식

### 4.1 실효 공격력 공식 (v14.3+)
```
eff_base = pool × (1 + ⓓ_정적) + ⓑ
resonance = pool × res% + res_flat
eff_atk  = [eff_base × (1 + Σ_dynamic_sum)] + resonance
```
- **ⓓ_정적**: 장비% + 여정% + 부옵션% (0.1625 + 0.01).
- **Σ_dynamic_sum**: AX 스택(8%*N), 스킬 유동 버프, 캐릭터 패시브 중첩(리디아 30%, 유미나 20%)의 **합산**.
- **v14.3 변경점**: 모든 유동 버프를 합산(Sum)한 뒤 기초 공격력에 곱하는 단일 곱연산 모델로 일원화.

### 4.2 최종 데미지
```
Damage = eff_atk × 스킬_계수 × (1 + DI + CR × CD) + Chain_Damage
```
- **DI**: 5개 여정의 피증 스택, 스킬 피증의 합산.
- **Chain_Damage**: 특정 여정(예: 유미나 카탈리스트)의 치명타 시 연쇄 피해 추가.
