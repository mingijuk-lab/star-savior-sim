이 문서는 시뮬레이션 엔진(`Core/calc_engine_v5.py`) 설계 및 대미지 검증을 위한 상세 계산 공식을 기술합니다.

> **최종 갱신**: 2026-04-03 (v16.0 — calc_engine_v5.py 전수 감사 기반)

## 1. 실효 공격력 계산 프로세스

### STEP 1: 기초 공격력 풀 (Raw Pool)
```
raw_pool = 기본_공격력 + 1250 (StellaArchive)
```
- **StellaArchive**: 모든 캐릭터에게 ATK +1250 고정 부여 (L122)

### STEP 2: 정적 공격력% 합산 (Static ATK%)
```
static_atk% = Σ(장비세트_ATK%) + Σ(여정_ATK%) + 부옵션%(0.1625 + 0.01)
```
- **부옵션 고정**: 6부위 × `$ATK$` + 전술 1% = 0.1725 (L132, L378)
- **패시브 ATK%**: `Modifier(StatType.ATK, passive%, PERCENT)` → 별도 permanent modifier (L139-140)
- ⚠️ 공명(Resonance)%와 StellaArchive는 이 합산에서 **제외**됨 (L374-376)

### STEP 3: 실효 기초 공격력 (eff_base)
```
eff_base = raw_pool × (1 + static_atk%) + 1000 (SystemConstant)
```
- **SystemConstant**: ATK +1000 고정 (L133, L380)

### STEP 4: 동적 버프 합산 (Dynamic Sum)
```
Σ_dynamic = Σ(turn_data[*_atk_p])       # JSON 턴 데이터의 공버프
          + AX_stacks[i] × 0.08          # AX 축복 스택
          + 리디아_stack × 0.06           # 캐릭터 패시브
          + 유미나_stack × (0.04|0.02)    # 캐릭터 패시브
          + 피의메아리_stack × 0.03       # 여정 동적
          + ...
```
- 모든 동적 ATK 버프를 **합산(Sum)**한 뒤 기초 공격력에 곱합니다 (L398-399).

### STEP 5: 공명 (Resonance)
```
resonance = raw_pool × res% + res_flat
```
- 공명은 곱연산 밖에서 **가산**됩니다 (L386-388, L399).

### STEP 6: 최종 실효 공격력 (eff_atk)
```
eff_atk = eff_base × (1 + Σ_dynamic) + resonance
```

## 2. 크리티컬 및 피해량 증가 계산

### 최종 치명타 확률 (eff_cr)
```
final_cr = char.get_stat(CR)
         + turn.cr_buf               # JSON 턴 버프
         + cr_from_buff              # 프레이 특수기 (0.30)
         + claire_cr_stack × 0.10    # 클레어 패시브
         + ra_cr_stack × 0.05        # 깊은 애도 여정

eff_cr = min(1.0, final_cr)          # 100% 캡
```

### 최종 치명타 피해 (eff_cd)
```
final_cd = char.get_stat(CD)
         + turn.cd_buf               # JSON 턴 버프
         + caster_cd_stack × 0.10    # 하늘의 심판 여정
         + assera_cd_stack × 0.06    # 아세라 패시브
```

### 피해량 증가 (total_DI)
```
total_DI = turn.di                   # 스킬 고유 DI
         + Σ(turn.*_di)             # JSON 동적 DI 필드들
         + m_di                      # 로자리아 업화 DI 등
         + omega_boost               # 허수의 개척자 (spec만)
         + fx_carry                  # FX 축복 캐리
         + 점화DI                    # 샤를(바니걸): attr_stack × 0.05
```

## 3. 방어율 계산 (v14.2+)

### DEF 계산 프로세스
```
target_def_base = 3000.0

# 방어력 감소 (Debuff — 합산)
current_def_red = 0.30 if smile_def_red_turns > 0 else 0.0

# 방어력 관통 (Character Stat + Turn Buff)
def_pen_total = char.get_stat(DEF_PEN) + turn.def_pen_buf

# 유효 방어력
effective_def = target_def_base × (1 - current_def_red) × (1 - def_pen_total)

# 방어율 (Defense Multiplier)
def_multiplier = 1.0 - effective_def / (effective_def + 3000.0)
```

### 방어율 조견표

| 유효 방어력 | 방어율 | def_multiplier | 비고 |
| :--- | :--- | :--- | :--- |
| 3000 | 50.0% | 0.500 | 기본 |
| 2100 (30% 감소) | 41.2% | 0.588 | 스마일 궁극기 적용 |
| 1500 | 33.3% | 0.667 | 50% 감소 |
| 0 | 0.0% | 1.000 | 완전 관통 |

## 4. 최종 데미지 공식

### 4.1 턴 데미지 (Turn Damage)
```
hr_prob = 1.0 if (프레이 && HR≥5 && 기본기) else 0.0

turn_dmg = (eff_atk × coeff × (1 + total_DI + cr × cd + hr_prob) + chain_dmg) × def_multiplier
```

- **hr_prob**: 프레이 하이롤 발동 시 추가 곱 +1.0 (실질 데미지 2배)
- **chain_dmg**: 현재 유미나 카탈리스트 등 연쇄 피해 (엔진에서 별도 구현)

### 4.2 최대 타격(Peak Hit) 데미지
```
peak_turn_dmg = (eff_atk × coeff × (1 + total_DI + 1.0(CD) + 1.0(HR)) + chain_dmg) × def_multiplier
```
- 모든 히트가 **크리티컬(cr=1)** + **HR 발동(hr=1)**이라고 가정한 이론적 최댓값.

### 4.3 로자리아 추가 기본기 데미지
```
# 업화 3+ 발화 시 추가 기본기 (L448-451)
ros_extra = eff_atk × 1.50 × (1 + total_DI + cr × cd) × def_multiplier  # 평균
ros_peak  = eff_atk × 1.50 × (1 + total_DI + cd)      × def_multiplier  # 피크
```

### 4.4 DPS 계산
```
DPS = Σ(turn_damage) / Σ(cycle_time)
```
- `cycle_time`: 각 턴의 소요 시간 합계
- 추가턴/추가타는 **시간 0**이므로 DPS를 높이는 요인

### 4.5 턴 시간 계산
```
if is_extra_attack:
    turn_time = 0.0  (cycle_time에 미반영)
elif is_extra_turn:
    turn_time = 0.0
else:
    eff_ga = min(ga_red_carry, 0.99)  # 음수 방지
    turn_time = (1000 / final_speed) × (1 - eff_ga)

cycle_time += turn_time
ga_red_carry = 0.0  # 소비
```

## 5. 상태이상 (DOT) 대미지 계산

도트 대미지는 기본적으로 스킬 대미지 계산식을 따르되 아래의 특수 규칙을 적용합니다.

### 디버프별 계수
- **연소**: 공격력 계수 0.2
- **출혈**: 공격력 계수 0.25
- **빙결**: 공격력 계수 0.15
- **중독**: 생명력 계수 0.05

### 특이사항
- **버프 적용**: 스탯 증가 버프(공격력, AX 스택 등) 및 DI가 DOT 대미지에 동일 적용 (v14.3 통합 모델)
- **AX 초기화**: 궁극기 직후 AX 스택 0 초기화 (데미지 계산 시점엔 기존 스택 유지)
- **관통**: 보호막(Shield)을 무시하고 HP에 직접 대미지 전달

## 6. 게임 공식 참조 (원본 4단계 공식)

### STEP 1: 기본 대미지 (Base Damage)
$$ \text{Base} = \frac{\left( \text{Atk} \cdot \text{Atk\_Coeff} + \text{HP} \cdot \text{HP\_Coeff} + \text{Def} \cdot \text{Def\_Coeff} \right) \cdot \left( 1 + \frac{\text{Spd}}{100} \cdot \text{Spd\_Coeff} \right) \cdot 1.85}{\text{Target\_Count}} $$
- **대미지 상수**: 1.85
- **타겟 수**: 전체 공격기에 4 적용 (현재 엔진에서는 간소화)

### STEP 2: 방어율 및 1차 대미지
$$ \text{Def\_Rate} = \frac{\text{Target\_Def}}{\text{Target\_Def} + 3000} $$
$$ \text{1st\_Dmg} = \text{Base} \cdot (1 - \text{Def\_Rate}) $$

### STEP 3: 증감률 및 2차 대미지
$$ \text{2nd\_Dmg} = \text{1st\_Dmg} \cdot (1 + \text{Inc\_Rate} - \text{Red\_Rate}) $$
- **DI 구성요소**: 속성 상성(±5%), 스킬 DI, 크리티컬(+CD), 브레이크/노바(+15%), 속성 스택(+1%/스택)
- **대미지 감소율**: 최대 0.5 (회피 시 +0.5)

### STEP 4: 최종 대미지 (Final Damage)
$$ \text{Final} = \lfloor \text{2nd\_Dmg} \cdot \text{Random}(0.95 \sim 1.05) \cdot \text{CP\_Factor} \rfloor $$
- **Random**: 엔진에서는 평균값 1.0 사용 (기대값 시뮬레이션)
- **CP_Factor**: 전투력 비율에 따른 계수 (엔진에서는 1.0 가정)

## 7. 엔진 공식 vs 게임 공식 차이점

| 항목 | 게임 원본 | 엔진 구현 | 이유 |
|------|----------|----------|------|
| 대미지 상수 1.85 | 적용 | 미적용 (계수에 내재) | 비교 분석이 목적이므로 상위 수치 단순화 |
| AoE 감쇄(/4) | 적용 | 미적용 | 단일 타겟 DPS로 비교 |
| 속도 계수 | `(1+Spd/100×Coeff)` | 행동 주기에 반영 | 데미지가 아닌 턴 간격으로 속도 영향 모델링 |
| 랜덤 (0.95~1.05) | 적용 | 1.0 고정 | 기대값 시뮬레이션 |
| CP Factor | 적용 | 1.0 고정 | 동일 전투력 가정 |
| 크리티컬 | 발생/미발생 분기 | `CR × CD` 기대값 | 확률 기반 평균 데미지 |
