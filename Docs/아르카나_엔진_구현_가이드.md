## 3. 캐릭터별 특수 메커니즘 ↔ 엔진 변수 매핑

### 3.1 프레이 (`is_frey`)

| 메커니즘 | 엔진 변수 | 초기값 | 증가 조건 | 캡 | 소비/리셋 | 효과 |
| :--- | :--- | :--- | :--- | :--- | :--- | :--- |
| 냉각(Cooling) | `attr_stack` | 0 | 기본기 +1, 특수기 +3 | 5 | 5 도달 시 0 리셋 | — (직접 데미지 효과 없음) |
| HR(하이롤) | `frey_hr` | 0 | 궁극기 +3, 달속성파티 +3×hr_chance | 5 | 5 도달 + 기본기 시 소비 | hr_prob=1.0 (DI +100%) |
| 치확 버프 | `frey_cr_turns` | 0 | 특수기 시 = 3 | — | 매 턴 -1 | CR +30% |
| 달속성 AG | `ga_red_carry` | — | 달속성파티: +0.24/턴 | — | — | AG 감소 |

**핵심**: HR과 냉각은 **별개** 변수로 독립 추적됩니다.

### 3.2 로자리아 (`is_rosaria`)

| 메커니즘 | 엔진 변수 | 증가 조건 | 캡 | 소비/리셋 | 효과 |
| :--- | :--- | :--- | :--- | :--- | :--- |
| 격동(Attribute) | `attr_stack` | 기본기 +1 | 5 | — | 업화 확률 기반 (attr × 0.20) |
| 업화(Ignition) | `ros_ign_stack` | 기본기: attr×0.20 누적, 궁극기: +3 | 5 | 특수기 시 소비 | 1+→DI+10%, 2+→DI+20%, 3+→DI+20%+추가기본기 |

### 3.3 유미나 (`is_yumina`)

| 메커니즘 | 엔진 변수 | 증가 조건 | 캡 | 효과 |

## 4. 방어 시스템 (v14.2+)

```python
# L319-328
target_def_base = 3000.0
current_def_red = 0.30 if smile_def_red_turns > 0 else 0.0
def_pen_total = char.get_stat(StatType.DEF_PEN, []) + t.get("def_pen_buf", 0.0)
effective_def = target_def_base * (1.0 - current_def_red) * (1.0 - def_pen_total)
def_multiplier = 1.0 - (effective_def / (effective_def + 3000.0))
```

- **스마일 궁극기**: `smile_def_red_turns = 2` → 2턴간 DEF 30% 감소 (L482, L485)
- **DEF Penetration**: `StatType.DEF_PEN` (Character stat) + `def_pen_buf` (Turn data)

## 7. 엔진 수정 시 필수 체크리스트 (Audit Points)

### 7.1 스택 캡(Cap) 확인
- 레인저 깊은애도: **5회**
- 캐스터 하늘심판: **3회**
- 어쌔신 키라: **3회**
- 오메가: **5회**
- 피의 메아리: **5회**
- 프레이 HR: **5회**
- 프레이 냉각: **5회**
- 로자리아 업화: **5.0** (실수형)
- 동적 패시브: 각각 3~5회 (p_stacks 참조)

### 7.2 트리거 시점 확인
- **Turn-start형**: `claire_cr` → 조건 체크 없이 매 턴 +1
- **Action-start형**: `lydia_atk`, `yumina_atk` → 매 행동 +1
- **Skill-trigger형**: `assera_cd` → `is_spec` 조건부
- **Hit-trigger형**: `ra_cr_stack` → `is_basic` 조건부
- **Kill(Consumption)형**: `frey_hr` → `is_basic and frey_hr >= 5` 조건부

### 7.3 턴 데이터(JSON) 필드 체크
| 필드 | 용도 | 기본값 |
|------|------|--------|
| `coeff` | 스킬 계수 | 1.5 |
| `extra_coeff` | HR 추가 계수 (hr_확률 곱연산) | 0 |
| `di` | 스킬 고유 DI | 0.15 |
| `*_di` | 추가 DI (키 접미사 매칭) | — |
| `*_atk_p` | 추가 ATK% (키 접미사 매칭) | — |
| `atk_buf` | 스킬 공격력 버프 | 0.0 |
| `cr_buf` | 스킬 치확 버프 | 0.0 |
| `cd_buf` | 스킬 치피 버프 | 0.0 |
| `def_pen_buf` | 방관 버프 | 0.0 |
| `spd_mult` | 속도 배율 | 1.0 |
| `ag_boost` | AG 감소율 | 0.0 |
| `is_ult` / `is_basic` | 스킬 타입 | false |
| `is_extra_attack` | 추가타 (시간 0) | false |
| `is_extra_turn` | 추가턴 (시간 0, CD 감소) | false |
| `hits` | 타격 횟수 (궁극기 기본 4) | 1 or 4 |
| `omega_elig` | 오메가 DI 적용 여부 | false |
| `note` | 감사용 설명 | "" |

### 7.4 고정밀 타이밍 (v15.1+)
- **AG 누적**: `ga_red_carry`는 항상 **합산(`+=`)** 처리. 패시브와 JSON 보너스가 중첩되어야 함.
- **추가 턴**: `is_extra_turn: true` → `turn_time = 0.0`
- **추가타**: `is_extra_attack: true` → `cycle_time`에 영향 없음

### 7.5 테스트 검증
수정 후 반드시 `verify_engine.py`를 실행하여 프레이(달속성파티), 클레어(바니걸) 등의 DPS가 예상 범위 내인지 확인하세요.
