## 상황별 업데이트 가이드

### ① 새 캐릭터 추가할 때

1. **`Data/캐릭터_스펙_마스터.md`** → 맨 아래 JSON 블록 추가 (기본 스탯, 패시브, 스킬 계수)
2. **`python Tools/extract_characters.py`** → `Data/characters.json` 동기화
3. **`$env:PYTHONPATH="."; python Tools/update_rotations.py`** → `Data/사이클_로테이션_마스터.md` 자동 갱신
4. 시뮬 실행: `$env:PYTHONPATH="."; python automated_run.py`

> ⚠️ **주의사항**: 캐릭터에 고유 스택 메커니즘(업화, 냉각, 점화, 출혈 등)이 있다면:
> - `calc_engine_v5.py`의 `calculate_dps()` 함수에 해당 로직을 직접 구현해야 합니다.
> - `update_rotations.py`의 `generate_rotation()` 함수에도 해당 스택 관리 로직을 추가해야 합니다.

### ② 기존 캐릭터 수치 바뀌었을 때 (패치)

1. **`Data/캐릭터_스펙_마스터.md`** → 해당 캐릭터 섹션 수치 수정
2. **`python Tools/extract_characters.py`** → JSON 갱신
3. **`python Tools/update_rotations.py`** → 로테이션 갱신

### ③ 장비 부옵션(공, 속, 치확, 치피)을 바꿀 때

1. **`python -m Core.calc_engine_v5`** 실행 시 터미널 프롬프트에 원하는 수치 입력
2. 혹은 **`automated_run.py`**의 `get_interactive_substats` 람다를 수정 (고정 수치가 필요할 경우)

### ④ 새 장비 세트 효과 추가할 때

1. **`Data/equipments.json`** → `four_piece_sets` 또는 `two_piece_sets`에 새 항목 추가
2. 시뮬레이션 시 자동으로 모든 4+2 조합이 생성되어 결과에 반영됨

### ⑤ 새 여정(Journey) 추가할 때

1. **`Data/equipments.json`** → `journeys` 섹션에 새 항목 추가
   - `type`: 임의 식별자
   - `modifiers`: 정적 스탯 버프 목록 (`[{stat, val, type}]`)
   - `restrict`: 제한 조건 (`{"class": "레인저"}` 또는 `{"char": "유미나"}`)
   - `note`: 동적 효과 설명 (엔진에서 하드코딩 필요한 경우)
2. **동적 효과가 있는 경우**: `calc_engine_v5.py`에 해당 로직을 직접 구현

### ⑥ 새 축복(Blessing/Arcana) 추가할 때

1. **`Data/equipments.json`** → `arcana_blessing` 섹션에 새 항목 추가
   - `stat`: 적용 스탯 (`ATK`, `CRIT_RATE`, `CRIT_DAMAGE`, `SPEED`)
   - `base_val`: 기본 부여 수치
   - 동적 효과가 있으면 `calc_engine_v5.py`에 별도 로직 구현

### ⑦ HTML 대시보드 배포할 때

```bash
$env:PYTHONPATH="."; python automated_run.py
python generate_html.py
# index.html에 자동 복사됨 → git push 후 GitHub Pages 배포
```

## AI에게 시뮬 요청하는 방법

### 단일 캐릭터 DPS 계산

```
로자리아를 AX + 통찰4+투지2 조건으로 DPS 계산해줘.
```

### 전체 빌드 최적화

```
유미나와 로자리아 전체 여정×장비 조합 비교 표 만들어줘.
```

### 새 캐릭터 추가 후 시뮬

```
스펙 마스터 파일에 [캐릭터명] 추가했어. 전체 여정×장비 DPS 비교해줘.
```

### 부옵 성장 경로 분석

```
프레이(달속성파티)의 공격력% 부옵 0%~50% 성장 경로 분석해줘.
```
